# Handoff: TourMaster — система підбору та бронювання турів

## Overview
Повний дизайн-пакет інформаційної системи туристичної агенції «TourMaster»:
клієнтський веб-сайт (пошук, перегляд, бронювання турів) + адміністративна панель
(управління каталогом, бронюваннями, клієнтами). Усього 10 сторінок у двох viewport
(Desktop 1280px, Mobile 375px для ключових сценаріїв), а також design tokens та
бібліотека компонентів.

## About the Design Files
Файли в цьому пакеті — **дизайн-референси, створені в HTML** (інтерактивні прототипи,
що демонструють бажаний вигляд і поведінку), **а не production-код для прямого копіювання**.
Завдання — **відтворити ці макети в існуючому середовищі цільового кодбейсу**, використовуючи
його усталені патерни та бібліотеки.

Цільовий стек (узгоджений із замовником):
- **Frontend:** Angular 18, standalone components, TypeScript
- **UI-бібліотека:** PrimeNG (Carousel, DataView, Dialog, Toast, Rating, Galleria, Paginator)
- **Стилі:** Tailwind CSS поверх кастомної PrimeNG-теми
- **Стан:** Angular Signals + `CartService` (кошик бронювань)
- **Зображення:** `ng-lazyload-image` (lazy loading карток турів)

Тобто HTML тут — джерело істини щодо візуалу, відступів і копірайтингу; реалізувати слід
нативними Angular standalone-компонентами + PrimeNG, а кольори/типографіку винести в
Tailwind-конфіг та CSS-перемінні теми PrimeNG.

## Fidelity
**High-fidelity (hifi).** Фінальні кольори, типографіка, відступи, тіні, стани та
мікровзаємодії. UI слід відтворювати піксель-у-піксель засобами кодбейсу. Точні значення —
у розділах **Design Tokens** і **Screens / Views**.

## Component Library
Специфікації — у секції «Бібліотека компонентів» прототипу. Ключове:

- **Кнопки:** `primary` (синій fill, radius 12, padding 12×22), `secondary` (обведення
  1.5px primary), `danger` (червоний fill), `ghost` (прозора), `CTA` (помаранчевий fill,
  вага 700, тінь CTA). Hover: затемнення на 1 крок. Disabled: `#C5CBD3`.
- **Поля форм:** border 1.5px `line`, radius 12, padding 11×14. Focus: border `primary` +
  `box-shadow 0 0 0 3px primary-50`. Error: border `error` + ring `error-50` + текст помилки 12px.
  Success: border `success` + іконка ✓. → PrimeNG `InputText`, `Calendar`, `MultiSelect`,
  `Dropdown`, `Slider (range)`. Чіпи multiselect: pill `primary-50`/`primary-700`.
- **Бейджі статусів бронювання (5):** Нове = `warning-50`/`#92610A`; Підтверджено =
  `primary-50`/`primary-700`; Оплачено = `success-50`/`#157A3C`; Завершено = `#EEF0F3`/`#4B5563`;
  Скасовано = `error-50`/`#B42323`. Кожен — крапка 8px відповідного насиченого кольору + текст.
  **Стан розрізняється кольором, не лише текстом.**
- **Бейджі тур-картки:** «🔥 ХІТ» (secondary), «−25% АКЦІЯ» (error), «НОВИНКА» (success),
  «⏳ ОСТАННІ МІСЦЯ» (білий + бордер).
- **Toast (4):** success/error/warning/info — біла картка, лівий бордер 4px кольору типу,
  кругла іконка, заголовок 14/700 + підзаголовок 13. → PrimeNG `Toast` + `MessageService`.
- **Картки:** тур-картка (hover: `translateY(-4px)` + `sh-3`, transition .2s), booking-картка
  (фото 88–104 + інфо + статус-бейдж + дії), stat-картка (лейбл + іконка-плитка 32–34 + число
  900 + дельта).
- **Порожні стани:** кругла іконка-плитка 96px на `primary-50`/`secondary-50`, заголовок 18/800,
  опис, CTA-кнопка.
- **Скелетони:** shimmer-градієнт `linear-gradient(90deg,#EDEFF2 25%,#F6F7F9 37%,#EDEFF2 63%)`,

## Interactions & Behavior
- **Hover карток:** `transform: translateY(-4px)` + тінь `sh-1 → sh-3`, `transition: transform .2s, box-shadow .2s`.
- **Кнопки hover:** затемнення фону на 1 крок (`primary→primary-600`, `secondary→secondary-600`).
- **Вкладки:** активна отримує колір primary + нижній бордер 2.5px; transition кольору .15s.
- **Wizard прогрес-бар:** анімація ширини заповненої лінії при переході між кроками.
- **Карусель/Galleria:** свайп (mobile), стрілки + дотси (desktop), fullscreen у Galleria.
- **Поля:** focus-ring (`0 0 0 3px primary-50`); error-стан з повідомленням; success з ✓.
- **CTA «Забронювати»:** завжди видима — sticky sidebar (desktop), floating bottom-бар (mobile).
- **Toast:** автозникнення; типи success/error/warning/info через `MessageService`.
- **Доступність:** WCAG 2.1 AA, контраст ≥ 4.5:1, повна доступність з клавіатури, видимий focus, hit ≥ 44px.

## State Management (Angular Signals)
- `searchCriteria` (напрямок, дати, туристи) — форма пошуку, передається в каталог через query-params/signal.
- `catalogFilters` (країни[], діапазон цін, зірки[], харчування[], тривалість, сортування, сторінка) — фільтри DataView.
- `viewMode: 'grid' | 'list'` — перемикач каталогу.
- `tourDetails(id)` — резолвер/сигнал поточного туру; обрана дата + к-сть туристів → перерахунок ціни (`computed`).
- `bookingWizard` (step 1–5, дані туристів[], обрані послуги[], платіж) + `CartService` (підсумок, знижка, разом).
- `cabinet` (вкладка, фільтр бронювань, список бронювань, wishlist).
- `auth` (режим вхід/реєстрація, користувач, токен).
- **Admin:** dashboard-метрики, tours CRUD + стан Dialog, bookings (фільтр, обране, зміна статусу + сповіщення), clients (пошук, обраний клієнт, нотатки).
- Дані турів/бронювань/клієнтів — через сервіси з API; картки турів — `ng-lazyload-image`.
